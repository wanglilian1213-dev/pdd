# 写作质量固定题库

这里放可复查的极端题和答案卡，避免把“口头说测过”当成“有证据证明通过”。

当前文件：

- `fixed-fixtures.json`：110 个固定答案卡，覆盖普通写作、作业说明、评分标准、引用、恶意材料、数据、图表、医学、工程、格式、交付、AI 检测、降 AI、文章修改、文章评审、隐私和组合端到端场景。
- `red-team-scenarios-2026-05-26.json`：30 个由挑剔用户代理补出的偏门场景，覆盖医学、工程、数据、图表、地图、法律、人文、伦理、金融、格式和冲突要求。
- `seed-fixtures.json`：9 个早期最高风险种子题，保留为扩展模板和历史对照。

当前边界：

- `fixed-fixtures.json` 已有自动检查：编号必须是 `WQ-001` 到 `WQ-110`，关键字段不能为空，高风险类别不能缺，三路智能体补出的边缘场景不能被删。
- `red-team-scenarios-2026-05-26.json` 已有自动检查：关键领域、风险等级、用户要求、期望处理和失败红线不能为空。
- 它证明的是“固定题库和红队清单本身完整、可复查”，不是“110 个线上任务和 30 个红队任务都已经跑完”。
- 每个题都必须能追到自动测试、线上任务编号、下载文件或人工复核记录之一，才可以在审计表里写成“已证明通过”。

后续扩展标准：

- 每个题保留输入材料、用户要求、预期行为、失败红线。
- 真实线上复测时补任务编号、下载文件、截图或日志位置。
- 数据题补标准结果。
- 图表题补期望图数、图类型、关键标签和 Word/PDF 渲染检查结果。
- 引用题补正文观点、参考文献、来源位置和支撑理由。

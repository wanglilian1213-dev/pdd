# UltraQA Matrix: Writing Extreme Cases 2026-05-27

## Goal

Run a broad adversarial test pass for the pindaidai writing system. The target is not only whether pages open, but whether final academic writing can survive extreme user requests without fake analysis, unsafe professional details, bad charts, broken citation/format rules, prompt injection, or misleading success states.

## Safety Bounds

- Do not print secrets or inspect browser cookies/password stores.
- Do not create production paid tasks unless the path includes cleanup and enough account credit authority.
- Prefer local automated tests and temporary harnesses for broad coverage.
- Treat production upload-to-download flow as unproven unless a real task is created, completed, downloaded, opened, inspected, and cleaned up.
